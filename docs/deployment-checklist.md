# 배포 체크리스트

AWS EC2 웹 서버 구성을 만들거나 점검할 때 사용하는 체크리스트입니다.

## 인스턴스 생성 전

- [ ] 리전 선택: `ap-northeast-2`
- [ ] VPC 생성: `my-vpc`, `10.0.0.0/16`
- [ ] 서브넷 생성: `my-sub01`, `my-sub02`
- [ ] 인터넷 게이트웨이 생성 및 VPC 연결
- [ ] 라우팅 테이블에 인터넷 게이트웨이 경로 추가
- [ ] 서브넷을 라우팅 테이블에 연결
- [ ] 퍼블릭 IPv4 자동 할당 활성화
- [ ] 적절한 AMI 선택
- [ ] 인스턴스 유형 선택: `t3.micro`
- [ ] 키 페어 생성 또는 선택
- [ ] 스토리지 설정
- [ ] 보안 그룹 규칙 설정
- [ ] 필요한 태그 추가

## 인스턴스 생성 후

- [ ] 인스턴스 상태가 `Running`인지 확인
- [ ] 상태 검사가 통과되는지 확인
- [ ] SSH 접속 확인
- [ ] 웹 서버 설치 및 실행
- [ ] 브라우저에서 웹 페이지 접속 확인
- [ ] `my-web01` 기준 AMI 생성
- [ ] AMI를 사용해 `my-web02` 생성
- [ ] 각 서버의 웹 페이지가 서로 다르게 표시되는지 확인

## 고가용성 실습

- [ ] Application Load Balancer 생성: `my-alb`
- [ ] ALB 보안 그룹 생성: `my-alb-sg`
- [ ] 대상 그룹 생성: `webserver-tg`
- [ ] `my-web01`, `my-web02`를 대상 그룹에 등록
- [ ] 웹 서버 보안 그룹에서 HTTP 소스를 ALB 보안 그룹으로 제한
- [ ] ALB DNS 이름으로 접속 테스트

## S3 연동 실습

- [ ] IAM 역할 생성: `myS3AccessRole`
- [ ] EC2 인스턴스에 IAM 역할 연결
- [ ] EC2에서 S3 버킷 목록 확인
- [ ] EC2에서 S3로 파일 업로드
- [ ] S3에서 EC2로 파일 다운로드
